Getting Started with zenCSS alpha

An open-source website authoring and design toolkit that helps you build, style, and debug websites. Whether harnessing your mastery of a code editor or just beginning your HTML and CSS journey, you're about to build beautiful solutions with our tools. Let's dive in.

What's included?

Framework/Design System

Ensure your content displays beautifully. 6 UI skins, branded themes, automatic formatting of content/ text, testing/debugging toolkit, and dark mode.

Utility Toolkit

Over 100 utility classes that allow you to tweak, shape, test, debug and beautify your core design. Customize type, color and form to your heart's content.

UI content library

A collection of off-the-shelf UI components you use immediately to build websites better and faster. Soon, a marketplace featuring top designers.

Getting Started

Follow the simple setup steps here to activate our toolkit on your website. Then, any time you wish to work with zenCSS you will simply declare a container/row to house your HTML content. Everything stays encapsulated and automatically balances in perfect harmony in terms of color and form. Deploy alongside current solutions like Bootstrap without worry.

zenCSS uses a desktop-first approach

That is to say, you build website content for desktop, and zenCSS automatically accounts for it's presentation on tablet and mobile.

TLDR...

  • Containers - This is a section of your website. Think, layers of a cake. Each container is a "works great on all resolutions" box that handles logic for horizontal alignment on mobile, tablet and desktop devices. There is also a full width container - container-fluid.
  • Rows - Rows expand to 100% width of their parent and hold columns. They have no padding or margin. They have fancy pants flex logic applied to them so they "just work"- Of course the default behavior is modifiable using our custom classes.
  • Columns - Your content boxes - you put HTML objects in here. an h2? sure. Paragraph and an image? No problem. Want to skin them with a dark theme? We gotchu fam. Columns have margin and padding. Aligned top-left by default, automatically resizing and realigning to accomidate tablet/mobile.
  • We made Zentax, our own HTML micro-language

Let's start with a basic example using semantic HTML

                                                        

<main class="z-container" >
  <section class="z-row">
    <div class="z-col"> ... </div>
  </section>
</main>

It doesn't matter how many columns you place in a row (2-6 is standard), and it doesn't matter how many html items you place in a column - zenCSS sorts that out for you. Need to add more items to a given content area? Add another row or nest columns inside a column. Simple.

Alternatively, you may use Zentax, our shorthand micro-language, and write half the code NEW 

<z-container>
  <z-row>
    <z-col> ... </z-col>
  </z-row>
</z-container>

What the heck is Zentax?

The simplest language you'll ever learn. Actually you already know it. Zentax, our shorthand micro-language consists of only 3 tags, and is used for developing website content with zenCSS. We've created custom container, row and column tags to allow for cleaner markup.

You're free to continue using tradional HTML or try out Zenax for our box structure. Either works interchangably depending on which option works best for your project. In future versions, we'll have tools to compile Zentax to standard semantic HTML for deployment.

Dynamic, responsive text resizing

zenCSS includes a robust, dynamic, totally amazing, completely automatic, hands free, no subscription required, auto-working, smart response detecting - text scaling solution. This feature automatically scales your website's text to different font sizes at different viewport resolutions.

Out of the box, our standard text sizes are:

  • 14px on mobile
  • 16 px on tablet
  • 18 on desktop screens

The h1 - h6 tags automatically adjust at each break point based on the default font size specified. For the more technically inclined, if you right click and "inspect" an element below, then switch to the computed tab you will see the sizes change as your resize the window. (Note: you will need to have your inspect settings on responsive to view this activity)

UI Utilities

zenCSS includes 100+ modifier classes that you may use to modify the appearance of html elements. To demonstrate how they work, let's change the code example UI element of this website to use the brand theme. 

<pre class="default-primary">...</pre>

Remember, themes affect bg color and text color. And zenCSS being the awesome tool that it is lets you override either at the element level. The order is the theme, then any background modifications then any text modifications, then any other styling, animation, etc. Here, we're setting the default theme, then immediately customizing the view. Not that you should, but you can. And all without using !important. 

<pre class="theme-primary bg-secondary text-purple border-0">...</pre>

Controlling how content breaks

By defauilt zenCSS strives to make your items display and realign beautifully on all devices, but you can use our utilities take control at anytime to fine tune using our cusom classes. The default logic for zenCSS is to display three columns on desktop, two on tablet and one on mobile. If you had 6 children in a row, you could add the class="two-up" to the row, and your children would occupy 50% of the width. SImilarly, use four-up and each child will each take a quarter of the space.

We intend to use the fibonaci sequence to deliver .layout classes in future updates that will make your content display using the principles of this magical number. You may already notice it peppered throughout our framework.

Debugging your website just got a whole lot easier

In practice, debugging stylistic issues related to CSS is not as straightforward as debugging functional JavaScript issues using console.log(). And that's before we even consider mobile/tablet issues, pre-processors, lack of source maps, minification, etc.

With zenCSS "genie mode" we overlay outlines on your content so you can effortlessly pinpoint and fix the source of your bugs. This works in tandem with the inspect element property of your browser's dev tools to quickly isolate the cause of your issues. The purple and green dotted lines indicate the boundaries of your containers, with purple representing fluid containers. The blue solid lines delineate the rows, while red outlines encircle your columns for better visualization. Paragraphs have a thin orange outline. Soon, we'll add a big ol alignment grid. Installation

To install zenCSS on your website, Link to our css file in the head section of your HTML document, then include the zencss.js file right before the closing body. 

<link rel="stylesheet" href="path-to/zen.css">
<script src="path-to/zencss.js"></script>

Usage

There are 2 ways to use zenCSS. directly on the body (globally affecting the site),or on an html element.

Globally: Simply declare zen on your body. 

<body class="zen"> 
...
</body>

Mix in: Declare zen anywhere on your site to unlock its capabilities without affecting anything else already in production. 

<section class="zen"> 
...
</section>
When you write code inside a parent declared as "zen", you are considered "in zen" and can utilize all our tools and features. Return to where you were when you followed the simple setup steps.
