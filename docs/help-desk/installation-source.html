Setup your SASS Compiler

zenCSS offers a robust way to customize your styles using SCSS, a preprocessor scripting language that is interpreted or compiled into CSS. To get started, you'll first need to set up SASS, the compiler for SCSS.

Installing SASS via NPM

Before proceeding, ensure that you have Node.js installed on your system. If not, visit the official Node.js website to download the latest version. Once Node.js is installed, you can install Dart SASS on your machine.

Installation:

Open your terminal or command prompt and execute the following command to install SASS:

npm install sass

Navigate to the SCSS Directory:

Change your current directory to where the SCSS files are located:

cd path-to-your-project/scss

Compile SCSS to CSS:

You can compile your SCSS files into CSS using the following commands:

For compressed CSS output, use:

sass --style=compressed zen.scss:../dist/zen.min.css

For expanded CSS output, use:

sass --style=expanded zen.scss:../dist/zen.css

To watch the folder and autogenerate CSS output, use:

sass --style=expanded zen.scss:../dist/zen.css --watch

Remember, every time you make changes to the SCSS files, you'll need to recompile them to see the changes reflected in your project.

Setup Webpack for JavaScript modules

Introduction

This guide outlines the steps to set up Webpack for bundling JavaScript modules in the zenCSS project.

Prerequisites

  • Node.js installed on your system
  • npm (Node Package Manager), typically bundled with Node.js

Installation and Configuration

Step 1: Install Webpack and Webpack CLI

Install Webpack and the Webpack CLI as development dependencies in your project.

npm install webpack webpack-cli --save-dev

Step 2: Configure webpack.config.js

When configuring your webpack.config.js, set up separate entry points for the zenCSS core modules and your custom modules.

This approach allows you to keep the original source intact and manage your custom scripts separately. Here's an updated example configuration:


const path = require('path');

module.exports = {
entry: {
zenCore: ['./js/modules/core-module1.js', './js/modules/core-module2.js'], // Core zenCSS modules
custom: ['./js/custom/custom-module1.js', './js/custom/custom-module2.js'] // Your custom modules
},
output: {
filename: 'zencss.bundle.js', // This will create separate bundles for core and custom modules
path: path.resolve(__dirname, 'dist/js'),
},
mode: 'development' // Change to 'production' for minified output
}

To run:

npx webpack

Autoprefixer Recommendations

Autoprefixer is a crucial tool for handling CSS vendor prefixes to ensure cross-browser compatibility. It automatically adds the necessary prefixes to your CSS properties to make your styles work correctly across various browsers.

Installation:

You can install Autoprefixer as part of your SASS workflow. Here are the steps to set it up:

  1. Install Autoprefixer as an NPM package: Open your terminal or command prompt and execute the following command: 
    npm install autoprefixer
  2. Configure Autoprefixer in your SASS compilation process: Depending on your SASS compiler, you can configure Autoprefixer to run as part of your compilation process. Here's an example of how to add Autoprefixer to your SASS command: 
    sass --style=expanded --watch input.scss output.css --load-path=node_modules
    Make sure to replace "input.scss" with your input SASS file and "output.css" with your desired output CSS file. The "--load-path=node_modules" flag ensures that Autoprefixer is loaded from your NPM packages.

With Autoprefixer integrated into your SASS compilation, you can write CSS without worrying about manually adding browser-specific prefixes.

Customizing zenCSS

zenCSS provides a robust user settings file allowing control over everything from the color system to your preferred border radius setting. It's worth noting any changes made to our source files will be lost in future software updates. To safely customize your settings and ensure they are preserved across updates to the zenCSS framework, follow these steps:

  1. Create a Custom Settings File: Create a new file named custom-settings.scss in a separate directory, such as a custom directory within your project's root.
  2. Set Your Custom Variables: Inside custom-settings.scss, define your custom variables such as colors, fonts, and others. For example: 
    $primary-color: #yourColorValue;
  3. Import Custom Settings: In the main SCSS file (zen.scss), import your custom-settings.scss right after the default settings to override them. Ensure the path to your custom file is correct based on its location. For example: 
    @import 'core/user-settings.scss';
@import '../custom/custom-settings.scss'; // Path to your custom overrides
// ... rest of the SCSS imports
  4. Review Documentation: Always refer to the official documentation for detailed examples and guidelines on creating custom variables.
  5. Use Version Control: Employ a version control system like Git to manage your customizations, which will also facilitate easy updating when new versions of the framework are released.

Following these steps will allow you to customize the framework to fit your needs while keeping your changes intact through updates.
