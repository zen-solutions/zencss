Customizing zenCSS: A User’s Guide

zenCSS is engineered to be a comprehensive yet flexible toolkit for modern web development. While the library comes with a well-thought-out set of styles, we understand the necessity for customization to meet specific project requirements. This document outlines the workflow for customizing zenCSS classes without modifying the original SCSS files.

How to override zenCSS

Begin by integrating zenCSS into your project. Depending on your preference, you can either link the compiled CSS file in your HTML or import the main SCSS file if you're using SCSS. 

<!-- If using CSS -->
<link rel="stylesheet" href="path-to/zen.css">
// If using SCSS
@import 'scss/zen.scss';

Creating Custom Styles

Create a new CSS or SCSS file for your custom styles. In this file, you can override the styles of zenCSS by specifying new rules for the classes you wish to customize. 

// custom-styles.scss
.class-to-override {
    property: new-value;
}

Importing Custom Styles

Ensure your custom styles are loaded after zenCSS so that your overrides take precedence. In the case of CSS, add a new link element after the link to zenCSS. In the case of SCSS, import your custom styles file after importing zenCSS. 

<!-- If using CSS -->
<link rel="stylesheet" href="path-to/zen.css">
<link rel="stylesheet" href="custom-styles.css"> 

// If using SCSS
@import 'scss/zen.scss';
@import 'custom-styles.scss';

Inserting SCSS into the zenCSS build

You may want to override specific files in the zenCSS build file by specifying overrides for the classes you wish to customize. Note that these will be overwritten in future updates so you will need to reapply these modifications if you choose this route. 

// in zen.scss
@import 'user-settings.scss';
@import 'your-user-settings-overrides.scss';

Compiling SCSS (if necessary)

If you're working with SCSS, compile your SCSS files into CSS using a tool like Sass, Gulp, or Webpack.

Testing and Iteration

Test your overrides in the browser to ensure they are working as intended. Iterate on your custom styles, re-compiling your SCSS as necessary, until you achieve the desired aesthetics and functionality.

Additional Best Practices

Maintain a separate file for overriding styles to keep customizations organized and easier to manage. This separation also facilitates smoother updates to zenCSS in the future without losing your customizations.

Your workflow may necessitate you inserting override files into the zen.scss build script. This is understandable but not ideal as In future updates, these declarations may be overwritten so you would need to redo them.

As you venture into customizing zenCSS, this structured approach will ensure a streamlined process, aligning with the efficient and precise ethos of the zenCSS toolkit.
