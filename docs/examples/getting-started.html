Getting started with zenCSS

Introduction

Welcome to zenCSS, your flexible and intuitive CSS framework. Whether you're a seasoned developer or just starting, zenCSS is designed to simplify your web design process and enhance your projects.

Quick Start

  • Download or clone the main branch of zenCSS.
  • Explore the hello-world.html in docs/examples to get a feel of zenCSS in action.
  • Or quickly experiment with our sandbox for a hands-on experience.

Installation

Downloading zenCSS

Clone or download the main branch from our repository.

File Structure Overview

Familiarize yourself with the core files:

Manual Installation

  1. CSS Linking: Add the zenCSS stylesheet in your HTML's head section. 
    <link rel="stylesheet" href="path-to/zencss.css">
  2. JavaScript Inclusion: Include the zencss.js script before the closing </body> tag. 
    <script src="path-to/zencss.js"></script>

Usage

Applying zenCSS

You can integrate zenCSS either globally or selectively:

  • Globally: To apply zenCSS styles throughout your website, add the zen class to the <body> tag. 
    <body class="zen"> 
    <!-- Your content here -->
</body>
  • Selectively: For specific sections, use the zen class on individual elements. This method allows zenCSS styling without affecting the rest of your website. 
    <section class="zen"> 
    <!-- Your content here -->
</section>

Working with zenCSS

We suggest using the following structure where you wrap your page content in a container-fluid. This way you can use container or container-fluid interchangably on any section of the page. The page-name class is optional but useful to target specific pages. The site-section class is also optional and sets spacing between sections. The default is 2em at the bottom of every site-section. This value is defined in user-settings.scss, overwrite as needed.

Basic Layout Example:

<main class="z-container-fluid page-name">
  <section class="z-container site-section">
    <div class="z-row">
      <div class="z-col"> ... </div>
    </div>
  </section>
</main>

Zentax Shorthand Syntax: Use our streamlined syntax for quicker coding. A shortcut user snippet file and a converter for semantic HTML are included.

<z-container-fluid class="page-name">
  <z-container class="site-section">
    <z-row>
      <z-col> ... </z-col>
    </z-row>
  </z-container>
</z-container-fluid>

Working with SCSS Files

zenCSS offers a robust way to customize your styles using SCSS, a preprocessor scripting language that is interpreted or compiled into CSS. To get started, you'll first need to set up SASS, the compiler for SCSS.

Installing SASS via NPM

Before proceeding, ensure that you have Node.js installed on your system. If not, head to the official Node.js website and download the latest version. Once Node.js is set up, you can install SASS globally on your machine.

  1. Installation:

    Open your terminal or command prompt and execute the following command to install SASS globally:

    npm install -g sass
  2. Navigate to the SCSS Directory:

    Change your current directory to where the SCSS files are located:

    cd path-to-your-project/scss
  3. Compile SCSS to CSS:

    You can compile your SCSS files into CSS in two different styles – compressed or expanded.

    • For compressed CSS (minimized file size), use:

      sass --style compressed zen.scss ../css/zen.css

    • For uncompressed CSS (easier to read and debug), use:

      sass --style expanded zen.scss ../css/zen.css

Remember, every time you make changes to the SCSS files, you'll need to recompile them to see the changes reflected in your project.

Customizing zenCSS

zenCSS provides a robust user settings file allowing control over everything from the color system to your preferred border radius setting. It's worth noting any changes made to this file will be lost in future software updates. To safely customize your settings and ensure they are preserved across updates to the zenCSS framework, follow these steps:

  1. Create a Custom Settings File: Create a new file named custom-settings.scss in a separate directory, such as a custom directory within your project's root.
  2. Set Your Custom Variables: Inside custom-settings.scss, define your custom variables such as colors, fonts, and others. For example: 
    $primary-color: #yourColorValue;
  3. Import Custom Settings: In the main SCSS file (zen.scss), import your custom-settings.scss right after the default settings to override them. Ensure the path to your custom file is correct based on its location. For example: 
    @import 'core/user-settings.scss';
@import '../custom/custom-settings.scss'; // Path to your custom overrides
// ... rest of the SCSS imports
  4. Review Documentation: Always refer to the official documentation for detailed examples and guidelines on creating custom variables.
  5. Use Version Control: Employ a version control system like Git to manage your customizations, which will also facilitate easy updating when new versions of the framework are released.

Following these steps will allow you to customize the framework to fit your needs while keeping your changes intact through updates.
