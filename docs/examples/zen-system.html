The ZEN (Zero Effort Nomenclature) Ideoligy

ZEN, standing for Zero Effort Nomenclature, amalgamates BEM, utility-first CSS, and atomic design ideologies into a singular, modular CSS organization method.

TLDR: Style elements from the outside in. .theme-x .bg-x .text-x .any-other-style-stuff-here. Put type and state classes up front. .card .inactive .style-stuff-here

1. Elements, Components and Sections

  • We structure individual website elements such as an image or paragraph, inside components (z-col -> card or modal), inside a framework to develop sections of a website.
  • Each item is globally themable and/or modifiable at any level via background, text, style, and state properties.
    • Example: A paragraph and button element in an z-col card component, which generates the text portion of an image/text section

2. Modifiers

  • Classes used to modify elements, components, sections, or the zenCSS framework.
    • Order of operations: Type, State, Theme, Background, Text, Style.
    • Example: card inactive theme-dark bg-white text-black italic

3. State WIP

  • Active (default), Inactive, Hidden, Visible, Block, Inline-block, Inline, Flex.

4. Structural Layout

  • Elements/Components are placed within Columns.
  • Columns are organized in Rows within a Container.
  • Rows and/or containers are stacked to create horizontal sections of a web page.
  • Container-fluid is used for vertical section spacing, section backgrounds or full screen content areas.

ZEN facilitates a structured yet flexible styling convention, simplifying UI development while ensuring a cohesive design language.
