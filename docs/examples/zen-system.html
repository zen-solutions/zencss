The ZEN (Zero Effort Nomenclature) system

ZEN, standing for Zero Effort Nomenclature, amalgamates BEM, utility-first CSS, and atomic design ideologies into a singular, modular CSS organization method.

1. Elements and Components

  • Globally themable and/or modifiable via background, text, style, and state.
    • Example: A card component containing a paragraph and button element

2. Modifiers

  • Type, State, Theme, Background, Text, Style.
    • Example: card inactive theme-dark bg-white text-black italic

3. State

  • Active (default), Inactive, Hidden, Visible, Block, Inline-block, Inline, Flex.

4. Structural Layout

  • Elements/Components are placed within Columns.
  • Columns are organized in Rows within a Container.
  • Containers are stacked to create horizontal sections of a web page.
  • Container-fluid is used for vertical section spacing, section backgrounds or full screen content areas.

ZEN facilitates a structured yet flexible styling convention, simplifying UI development while ensuring a cohesive design language.
