Introduction

Welcome to the documentation for zenCSS. zenCSS is a custom CSS framework designed to provide a solid foundation for building user interfaces with ease and consistency. It offers a range of utilities, components, and themes to accelerate the development process while ensuring a visually appealing and user-friendly experience.

This documentation aims to provide a comprehensive guide to all the features, classes, and utilities available in zenCSS. Whether you are just getting started or looking to explore advanced features, this documentation will serve as a reliable reference.

Let's dive into the various aspects of zenCSS and explore how it can help streamline your frontend development workflow.

Experimental Classes

1. Multi-Column Layout Classes

These classes allow you to create flexible multi-column layouts. They utilize the flex container property to organize child elements into columns, with each class name indicating the number of columns in the layout.

  • .two-up: Creates a layout with 2 columns.
  • .three-up: Creates a layout with 3 columns.
  • .four-up: Creates a layout with 4 columns. (Redundant as this is default)
  • .five-up: Creates a layout with 5 columns.
  • .six-up: Creates a layout with 6 columns.
  • .seven-up: Creates a layout with 7 columns.
  • .eight-up: Creates a layout with 8 columns.

Each of these classes utilizes the flexbox layout model to arrange child elements into the specified number of columns. The flex property values for child elements ensure that they occupy the correct proportion of the container's width, thereby achieving the desired multi-column layout. Mobile/Tablet support is WIP.

2. Tablet Support

  • .reverse-on-tablet applied to z-col, this swaps two children for tablet. Useful for image/text when you want to control which shows first when stacking for tablet/mobile view.

FLEX

1. Flex Container Class

  • .d-flex:
    display: flex; - This class sets the display property to flex, making the element a flex container.

2. Flex Grow Classes

These classes are used to set the flex-grow property of flex items, which determines how much a flex item will grow relative to the rest of the flex items in the flex container.

  • .feature:
    flex-grow: 2 !important;
  • .feature-sm:
    flex-grow: 3;
  • .feature-md:
    flex-grow: 5;
  • .feature-lg (Note: .feature-lg is defined twice, the latter will override the former):
    flex-grow: 6; (This will be the effective flex-grow value due to CSS overriding rules.)

3. Order Modification Classes

These classes are used within a flex container to change the order of flex items.

  • .reverse > :nth-z-col(1):
    order: 2; - This sets the order of the first flex item to 2.
  • .reverse > :nth-z-col(2):
    order: 1; - This sets the order of the second flex item to 1.

4. Flex Expansion Class

  • .expand:
    flex: 1; - This class allows a flex item to grow and shrink in order to fit the space available in a flex container.

5. Alignment Class

  • .center-me - This class vertically centers a flex item within a flex container.

These classes provide various utilities for working with the CSS Flexbox layout model. They offer convenient ways to control the growth, order, and alignment of flex items within a flex container.

DESIGN

These styles provide a variety of visual treatments for buttons, from standard to ghost to squared buttons, as well as for inactive states and tout containers. The mixins and variables allow for easily adjustable and themeable styles.

Button Styles and Sizes:

These classes and mixins are tailored for styling and sizing buttons.

Variables:

  • $btn-color: $cta-color;
  • $btn-hover-color: darken($cta-color, 20%);

Mixin for hover styles:

  • @mixin btn-hover($bg-color, $border-color: $bg-color):
    When hovered over, changes the background-color and border-color of the element.

Button Classes:

  • .btn:
    Default button with background-color: $btn-color;.
  • .btn-ghost:
    A button with color: $btn-color;, border: 2px solid $btn-color;, and background: none;. On hover, it changes color, border, and background properties.
  • .btn-ghost-light and .btn-ghost-dark:
    Variants of .btn-ghost with different color schemes for light and dark themes respectively. They have customized hover styles.
  • .btn-invert:
    Button with background-color: $btn-color;.

Button Sizes:

  • .btn-xl:
    Extra-large button with min-width: 200px;, max-width: 200px;, font-size: 14px;, and min-height: 45px;.
  • .btn-lg:
    Large button with min-width: 150px;, max-width: 150px;, font-size: 14px;, and min-height: 45px;.
  • .btn-md:
    Medium button with min-width: 120px;, max-width: 120px;, font-size: 14px;, and min-height: 35px;.
  • .btn-sm:
    Small button with min-width: 75px;, max-width: 75px;, padding: .3em;, and font-size: 12px;.

Unique Button Styles:

  • .btn-square:
    A square button with a 3D effect and arrow indicators. It changes its position and border on click.
  • .inactive:
    An inactive state style with cursor: not-allowed;, opacity: 0.65;, and pointer-events: none;.

Touts:

  • .tout:
    A styled container for highlighting content with border-radius: 8px;, padding: 0.3em 1em;, and background: $accent-color;.
  • .tout-wrapper and .corner-top:
    Additional elements to style and position content within a .tout.

Padding & Margin Utilities Documentation

General Padding and Margin:

  • .p-0 to .p-5: Applies padding around an element. The padding size increases by 0.25rem for each level starting from 0.
  • .m-0 to .m-5: Applies margin around an element. The margin size increases by 0.25rem for each level starting from 0.

Specific Side Padding and Margin:

  • .mt-0 to .mt-5: Applies top margin to an element. The margin size increases by 0.75em for each level starting from 0.
  • .mb-0 to .mb-5: Applies bottom margin to an element. The margin size increases by 0.75em for each level starting from 0.
  • .ml-0 to .ml-5: Applies left margin to an element. The margin size increases by 0.75em for each level starting from 0.
  • .mr-0 to .mr-5: Applies right margin to an element. The margin size increases by 0.75em for each level starting from 0.
  • .pt-0 to .pt-5: Applies top padding to an element. The padding size increases by 0.75em for each level starting from 0.
  • .pb-0 to .pb-5: Applies bottom padding to an element. The padding size increases by 0.75em for each level starting from 0.
  • .pl-0 to .pl-5: Applies left padding to an element. The padding size increases by 0.75em for each level starting from 0.
  • .pr-0 to .pr-5: Applies right padding to an element. The padding size increases by 0.75em for each level starting from 0.

Extended Padding and Margin Classes:

  • .p-6: Applies 2em padding around an element.
  • .m-6: Applies 2em margin around an element.
  • .p-7: Applies 3em padding around an element.
  • .m-7: Applies 3vw margin around an element.
