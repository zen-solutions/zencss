Introduction

Welcome to the documentation for zenCSS. zenCSS is a custom CSS framework designed to provide a solid foundation for building user interfaces with ease and consistency. It offers a range of utilities, components, and themes to accelerate the development process while ensuring a visually appealing and user-friendly experience.

This documentation aims to provide a comprehensive guide to all the features, classes, and utilities available in zenCSS. Whether you are just getting started or looking to explore advanced features, this documentation will serve as a reliable reference.

Let's dive into the various aspects of zenCSS and explore how it can help streamline your frontend development workflow.

Experimental Classes

1. Multi-Column Layout Classes

These classes allow you to create flexible multi-column layouts. They utilize the flex container property to organize child elements into columns, with each class name indicating the number of columns in the layout.

.two-up: Creates a layout with 2 columns.

.three-up: Creates a layout with 3 columns.

.four-up: Creates a layout with 4 columns. (Redundant as this is default)

.five-up: Creates a layout with 5 columns.

.six-up: Creates a layout with 6 columns.

.seven-up: Creates a layout with 7 columns.

.eight-up: Creates a layout with 8 columns.

Each of these classes utilizes the flexbox layout model to arrange child elements into the specified number of columns. The flex property values for child elements ensure that they occupy the correct proportion of the container's width, thereby achieving the desired multi-column layout. Mobile/Tablet support is WIP.

2. Tablet Support

.reverse-on-tablet applied to z-col, this swaps two children for tablet. Useful for image/text when you want to control which shows first when stacking for tablet/mobile view.

3. Order Modification Classes

These classes are used within a flex container to change the order of flex items.

.reverse > :nth-z-col(1): order: 2; - This sets the order of the first flex item to 2.

.reverse > :nth-z-col(2): order: 1; - This sets the order of the second flex item to 1.

4. Flex Expansion Class

.expand: flex: 1; - This class allows a flex item to grow and shrink in order to fit the space available in a flex container.

5. Alignment Class

.center-me - This class vertically centers a flex item within a flex container.

These classes provide various utilities for working with the CSS Flexbox layout model. They offer convenient ways to control the growth, order, and alignment of flex items within a flex container.

Color Palettes and Utility Classes

These classes and Sass logic are designed to manage color schemes and create utility classes for background colors, text colors, and borders, among others.

Brand and Generic Color Palettes:

Two sets of color palettes: one for the brand (brand-color-variations) and another for generic usage (generic-color-variations).

Each palette has dark and light variations.

Border Classes:

.border: A class for a solid border with color #c2c2c2.

Button Classes (Continued from previous input):

.tout: A class for styled containers with border-radius, padding, and background color.

Misc:

Variables for border colors ($colors with 6 shades from light to dark).

Various button styles like .btn-ghost, .btn-invert, and .btn-square.

Button size classes like .btn-xl, .btn-lg, .btn-md, and .btn-sm.

These classes provide a broad range of utilities and styles to manage the appearance and color schemes of elements within a web project. The color palettes and maps offer a structured way to handle colors, while the utility classes provide quick ways to apply common styles. The button and border classes extend the styling options further, offering more variety and flexibility in design.

These classes are designed for on-the-fly troubleshooting, especially useful during the development phase to highlight elements on the page.

.test-1: Adds a 2px dotted outline around the element with a green color.

.test-2: Adds a 2px solid outline around the element with a red color.

.test-3: Adds a 4px solid outline around the element with a blue color.

.test-4: Adds a 2px solid outline around the element with an orange color.

.test-5: Adds a 2px solid outline around the element with a purple color.

Utilizing these test classes makes it easier to visually identify elements and their dimensions on the page, aiding in debugging and layout adjustments.
